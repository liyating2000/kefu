# PRD：渠道管理（第 1 部分）

> 本轮范围：**渠道列表（增删改查） + 渠道配置-客户端属性**  
> 版本：v0.1  
> 编写日期：2026-04-24  
> 说明：本 PRD 覆盖渠道管理模块中渠道实体的维护及"客户端属性"子模块。高频操作配置、满意度、转人工设置、问卷调研、参数设置等其他子模块将在后续轮次补充。

---

## 1. 文档概述

### 1.1 文档目的
定义渠道管理模块的渠道列表（增删改查）及客户端属性配置功能，为产品、研发、测试、UI 提供实现依据。

### 1.2 产品背景
科大讯飞在线客服系统支持客户通过 PC、APP、微信小程序等多种入口发起咨询。每种入口对应一个"渠道"，渠道是客服系统中对外服务入口的基本配置单元。管理员需要通过"渠道维护"完成渠道的注册、编辑、删除，并通过"渠道配置"对单个渠道的访客端主题外观、机器人、聊天功能组件等进行差异化定制。

### 1.3 目标用户
- **系统管理员**：负责渠道创建、编辑、删除及全量配置
- **运营管理员**：负责单渠道的客户端属性、组件开关、外观等日常维护

### 1.4 名词解释
| 名词 | 说明 |
|------|------|
| 渠道 | 客户访问在线客服的入口配置单元，如 PC 网页、APP、小程序等 |
| 渠道id | 系统为每个渠道生成的唯一标识 |
| 用户体系 | 渠道归属的租户/业务线用户体系 |
| 接入类型 | 客户端与客服系统的接入方式，如引入插件、授权链接等 |
| 渠道入口图标 | 访客端页面显示的客服入口 ICON |

## 4. 渠道列表（渠道维护）

### 4.1 需求描述
管理员通过渠道维护页查看、筛选、管理系统中所有已注册渠道，并作为进入"渠道配置"的统一入口。

### 4.2 页面入口
`网聊维护` Tab → `渠道维护` 二级 Tab

### 4.3 页面字段

#### 4.3.1 筛选区
| 字段名称 | 控件类型 | 是否必填 | 校验规则 |
|---------|---------|---------|---------|
| 渠道名称 | 文本输入框 | 否 | 最大 50 字符，支持模糊查询 |
| 渠道id | 文本输入框 | 否 | 支持精确查询 |
| 用户体系 | 下拉选择框 | 否 | 数据来源于用户体系配置 |
| 接入类型 | 下拉选择框 | 否 | 枚举：引入插件、授权链接、引用链接、引用插件等（以系统字典为准） |

#### 4.3.2 列表字段
| 列名 | 字段说明 |
|------|---------|
| 序号 | 当前页序号，从 1 开始 |
| 渠道名称 | 渠道的显示名称 |
| 渠道id | 渠道唯一标识，系统生成 |
| 用户体系 | 渠道所属用户体系 |
| 接入类型 | 显示接入类型枚举文本 |
| 创建时间 | 精确到秒，`YYYY-MM-DD HH:mm:ss` |
| 更新时间 | 最近一次修改时间，精确到秒 |
| 操作 | "编辑"、"删除"、"渠道配置" |

### 4.4 规则说明
- 列表默认按**更新时间倒序**展示
- 同一用户体系下**渠道名称不可重复**
- **渠道id 系统自动生成、全局唯一、不可修改**
- 存在关联数据（如已产生对话、配置）的渠道删除需二次确认（详见 4.7）
- 筛选条件之间为**AND**关系，点击"查询"触发检索，"重置"清空所有筛选并刷新

### 4.5 交互说明
- "新增"：点击弹出「新增渠道」弹窗（见 4.6）
- "查询"：按筛选条件检索
- "重置"：清空所有筛选项并回到默认结果
- "编辑"：点击弹出「编辑渠道」弹窗（见 4.8）
- "删除"：点击弹出二次确认弹窗（见 4.7）
- "渠道配置"：点击跳转到渠道配置页（默认进入"客户端属性"子 Tab，见第 5 章）
- 分页：底部展示页码、每页条数下拉、跳转输入框

### 4.6 新增渠道弹窗

#### 4.6.1 弹窗字段
| 字段名称 | 控件类型 | 是否必填 | 校验规则 |
|---------|---------|---------|---------|
| 渠道名称 | 文本输入框 | 是 | 最大 50 字符，同用户体系下不可重复 |
| 用户体系 | 下拉选择框 | 是 | 数据来源于用户体系配置 |
| 渠道入口类型 | 下拉选择框 | 是 | 枚举：PC、APP、小程序等（以系统字典为准） |
| 接入类型 | 下拉选择框 | 是 | 枚举：引入插件、授权链接、引用链接等；不同接入类型可能影响后续字段显隐 |
| 引用配置 | 下拉选择框 | 否 | 选项为系统内已有渠道名称；仅在特定接入类型（如"引入插件"/"引用链接"）下启用 |
| 渠道入口图标 | 图片上传 | 是 | 建议尺寸 64×64 px，支持 PNG 格式；限制大小 ≤ 500 KB |
| 渠道入口图标位置 | 下拉选择框 | 是 | 枚举：左下、右下、左上、右上等（以系统字典为准） |

> **待确认**：各枚举字典（渠道入口类型、接入类型、图标位置）的完整值域；"引用配置"的业务含义及触发条件。详见第 7 章。

#### 4.6.2 规则说明
- 必填项未填写时"确定"按钮保持禁用状态或点击后给出红色提示
- 渠道名称与同用户体系内重名时校验失败并提示"渠道名称已存在"
- 图标上传失败需提示具体原因（格式不符、大小超限、网络错误）
- 保存成功后关闭弹窗，刷新列表并定位到新记录

#### 4.6.3 交互说明
- "取消"：关闭弹窗，丢弃输入
- "确定"：校验通过后提交创建，成功后提示"新增成功"并刷新列表

### 4.7 删除渠道二次确认
- **标题**："删除渠道"
- **内容文案**：`确认删除渠道「{渠道名称}」？删除后该渠道下的配置将一并清除。`
- **按钮**："取消"、"确定"
- **交互**：点击"确定"后调用删除接口；成功提示"删除成功"，刷新列表；失败提示失败原因

> **待确认**：已产生历史会话/工单的渠道是否禁止删除？是否需要管理员二次密码校验？

### 4.8 编辑渠道弹窗

#### 4.8.1 弹窗字段
| 字段名称 | 是否可编辑 | 说明 |
|---------|----------|------|
| 渠道id | 否（只读） | 显示当前记录的渠道id |
| 渠道名称 | 是 | 同 4.6.1 校验规则 |
| 用户体系 | 是 | 同 4.6.1 校验规则 |
| 渠道入口类型 | 是 | 同 4.6.1 校验规则 |
| 接入类型 | 是 | 同 4.6.1 校验规则 |
| 引用配置 | 是 | 同 4.6.1 校验规则 |
| 渠道入口图标 | 是 | 同 4.6.1 校验规则 |
| 渠道入口图标位置 | 是 | 同 4.6.1 校验规则 |

#### 4.8.2 规则说明
- 打开弹窗时回显当前记录全部字段值
- 其余规则同"新增渠道"
- 保存成功后"更新时间"刷新为提交时间

#### 4.8.3 交互说明
- "取消"：关闭弹窗，丢弃修改
- "确定"：校验通过后提交更新，成功提示"编辑成功"并刷新列表

### 4.9 渠道入口图标上传组件
- 组件结构：标签「渠道入口图标」+ 图片占位框 + 建议文案「建议尺寸 64 px * 64 px」+ 「修改」链接
- 默认态：占位框为空
- 已上传态：占位框显示已上传缩略图，"修改"链接用于重新选择图片

## 5. 渠道配置 - 客户端属性

### 5.1 需求描述
管理员对单个渠道的访客端**主题外观、机器人信息、基础设置、聊天功能组件开关**进行配置，变更结果影响访客端实际展示形态。页面右侧提供实时效果预览，便于配置时所见即所得。

### 5.2 页面入口
渠道列表行操作 "渠道配置" → 左侧子导航「客户端属性」（默认选中）

### 5.3 页面布局
```
┌──────────────┬────────────────────────┬──────────┐
│ 子导航         │ 配置区（多分区）         │ 效果预览  │
│ - 客户端属性   │ [访客端主题与外观]       │ 手机样机  │
│ - 高频操作配置 │ [机器人设置]             │ 实时预览  │
│ - 满意度       │ [基础设置]              │          │
│ - 转人工设置   │ [聊天功能组件控制]        │          │
│ - 问卷调研     │                         │          │
│ - 参数设置     │ [保存] [重置]           │          │
└──────────────┴────────────────────────┴──────────┘
```

### 5.4 分区一：访客端主题与外观

#### 5.4.1 页面字段
| 字段名称 | 控件类型 | 是否必填 | 校验规则 |
|---------|---------|---------|---------|
| 主题色选择 | 预设色板 + HEX 输入框 | 是 | 可选预设色（11 个预设色），或输入合法 HEX（如 `#7545E8`） |
| 聊天背景选择 | 预设背景图/色 + HEX 输入框 | 是 | 可选预设背景或输入合法 HEX |
| 访客端顶部 Logo | 图片上传 | 否 | 建议 44×44 px，PNG 格式；限制大小 ≤ 500 KB |
| 窗口标题配置 | 文本输入框 | 是 | 默认 "智能客服在线咨询"，最大 20 字符 |

#### 5.4.2 规则说明
- 选中的主题色/背景色同步回填到 HEX 输入框；HEX 输入合法值后也应同步选中对应预设（或切换到"自定义"态）
- 预览区实时渲染所选主题色与背景、Logo、窗口标题
- Logo 上传失败提示格式、大小、网络原因
- HEX 输入需即时校验 `#RRGGBB` 格式

#### 5.4.3 交互说明
- 主题色预设：点击圆形色块即选中，选中态有勾选标识
- HEX 输入：失焦或回车时校验并应用
- Logo 上传：未上传态显示占位图标 + "上传图片"；已上传态显示缩略图 + "修改 Logo"链接

> **待确认**：主题色是否支持自定义（即输入预设外的任意 HEX）？11 个预设色的具体色值是否由产品预先确定？

### 5.5 分区二：机器人设置

#### 5.5.1 页面字段
| 字段名称 | 控件类型 | 是否必填 | 校验规则 |
|---------|---------|---------|---------|
| 机器人名称 | 文本输入框 | 是 | 最大 20 字符；示例 "小智助手V4" |
| 机器人 ID | 文本输入框 | 是 | 系统内唯一；示例 "BOUY-KHK_9098" |
| 机器人种类 | 下拉选择框 | 是 | 数据来源于机器人配置 |
| 机器人类型 | 下拉选择框 | 是 | 数据来源于机器人配置 |
| 机器人头像 | 图片上传 | 否 | 建议 44×44 px，PNG 格式 |

#### 5.5.2 规则说明
- 机器人 ID 不允许重复；是否可修改需二次确认
- 机器人头像上传成功后在预览区同步更新

> **待确认**：机器人 ID 是否由用户输入还是系统自动生成？机器人种类/类型的枚举项清单？

### 5.6 分区三：基础设置

#### 5.6.1 页面字段
| 字段名称 | 控件类型 | 是否必填 | 校验规则 |
|---------|---------|---------|---------|
| 业务类型 | 下拉选择框 | 是 | 数据来源于业务类型字典 |
| 路由模式 | 下拉选择框 | 是 | 数据来源于路由模式字典 |

> **待确认**：业务类型与路由模式的取值范围？两者是否存在互斥或联动关系？

### 5.7 分区四：聊天功能组件控制

#### 5.7.1 组件开关清单
| 开关名称 | 默认值 | 开启说明（图内文案原样） |
|---------|-------|------------------------|
| 转人工 | 开 | 开启将显示转人工按钮，访客可以直接联系客服 |
| 超链接 | 开 | 开启访客端将识别超链接 |
| 上传文件 | 开 | 开启后允许客户端上传文件 |
| 小智在线 | 关 | 开启后提供小智在线功能 |
| 机器人 | 开 | 开启后为客户提供机器人服务 |
| 机器人问题是否解决 | 开 | 开启后提供机器人问题是否解决功能，访客可以评价 |
| 留言 | 开 | 开启后允许客户端进行留言 |
| 强认证 | 关 | 开启后对访客进行强认证 |

#### 5.7.2 规则说明
- 每个开关为独立配置项，可独立开关
- "机器人"开关与 5.5 机器人设置分区的联动关系待确认（关闭机器人时，机器人设置是否需置灰？）
- 开关变更后需点击"保存"才持久化

> **待确认**：
> 1. 各开关之间是否存在依赖关系？如"机器人问题是否解决"是否依赖"机器人"开启？  
> 2. "小智在线"与"机器人"的差异？

### 5.8 全局操作

#### 5.8.1 按钮
| 按钮 | 交互 |
|------|------|
| 保存 | 校验所有必填与格式，通过则提交；成功后提示"保存成功"并刷新右侧预览；失败提示错误字段 |
| 重置 | 二次确认后还原到本次进入页面时的初始状态（非系统默认值）。Toast 提示"已重置" |

### 5.9 效果预览
- 右侧展示手机样机，实时渲染所选主题色、聊天背景、顶部 Logo、窗口标题、机器人头像、评价样式等
- 实时性：配置项变更时即时渲染（无需点保存也预览）；保存后固化为渠道真实配置

> **待确认**：预览是否要求"保存后"才渲染，还是输入时即刻预览？

### 5.10 顶部刷新按钮区
渠道维护页顶部可见的"刷新队列中员工缓存 / 刷新工作组队列列表 / 刷新系统参数缓存 / 刷新脏词库 / 刷新满意度 / 刷新属性配置缓存"属于**全局运维操作**，其中"刷新属性配置缓存"与客户端属性配置修改后的缓存同步相关。本 PRD 仅做入口记录，具体行为在"系统管理/缓存刷新"模块另述。
