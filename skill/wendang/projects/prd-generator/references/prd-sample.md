## 权限说明

- **系统管理员**：创建考核周期、考核指标、考核模板、考核计划（不限，我管的）
- **部门管理员**：创建考核计划，考核对象和评分人员仅"我管的"，查看计划所有评分人员的评分结果
- **评分人员**：绩效评分，查看我评分的结果
- **考核对象**：查看绩效结果

## 绩效指标管理

### 需求描述
管理员想要对部门成员绩效考核时，需要先创建考核的指标。点击"指标管理"菜单，进入绩效指标维护界面。界面可实现对绩效指标信息的增、删、改、查功能。

### 页面字段

| 字段名称 | 字段类型 | 是否必填 | 校验规则 | 说明 |
|---------|---------|---------|---------|------|
| 指标名称 | 文本 | 是 | 最大长度50字符，不允许重复 | 考核指标的唯一标识 |
| 指标分类 | 下拉选择 | 是 | 必须选择已存在的分类 | 指标的所属类别 |
| 指标定义 | 文本 | 是 | 最大长度500字符 | 详细说明该指标的评分标准 |
| 创建时间 | 日期时间 | 自动生成 | 系统自动填充 | 记录指标的创建时间 |

### 规则说明
- 有权限查看该菜单的用户可以进行指标维护。
- 指标名称在系统中必须唯一，不允许重复。
- 被考核模板引用的指标不允许删除，需先移除引用关系。

### 交互说明

**查询条件**：
- "指标名称"：录入框，手动录入文字，支持模糊查询

**界面功能**：
- 左侧目录结构，支持增删改查目录，在目录下新增指标
- "查询指标"：根据组合条件进行定位查询指标项，查询结果显示到列表中
- "新增指标"：点击弹出指标项配置窗口，实现指标新增功能
- "修改指标"：点击弹出指标项配置窗口，实现指标修改功能

**交互逻辑**：
- "确定"：点击校验必填项，未填则输入框下方红色小字提示"xx不可为空！"；校验通过保存数据并在列表回显
- "取消"：关闭弹窗，不保存数据
- "×"：关闭弹窗，不保存数据

## 考核模板

### 需求描述
管理员创建考核的指标后，需要将考核指标添加到考核模板，考核模板由多个指标组成，手动设置每个指标的权重和指标总分。

### 页面字段

| 字段名称 | 字段类型 | 是否必填 | 校验规则 | 说明 |
|---------|---------|---------|---------|------|
| 模板名称 | 文本 | 是 | 最大长度50字符，不允许重复 | 模板的唯一标识 |
| 指标列表 | 关联数据 | 是 | 至少添加一个指标 | 包含的考核指标及其权重 |
| 创建时间 | 日期时间 | 自动生成 | 系统自动填充 | 记录模板的创建时间 |

### 规则说明
- 有权限查看该菜单的用户可以进行考核模板维护。
- 考核计划绑定该模板，并且状态为"进行中"时不允许编辑。
- 有计划关联该模板时不允许删除。

### 交互说明

**左侧目录结构**：支持多级新增考核模板

**筛选条件**：
- "模板名称"：录入框，支持模糊查询

**操作栏**：
- "新增"：点击弹出新增弹窗
- "编辑"：点击弹出编辑弹窗，考核计划绑定该模板，并且状态为"进行中"时不显示该按钮
- "删除"：点击弹出删除弹窗，如果有计划关联该模板，无法删除，Toast提示"该考核模板被考核计划xx关联，删除失败！"，如果被多个考核计划关联，Toast提示"该考核模板被多个考核计划关联，删除失败！"，如果未被关联，则弹窗提示

### 逻辑说明
- "指标权重"：默认无数据需要手动填写，所有指标的总权重为100%
- "添加指标"：点击弹出添加指标弹窗，支持按层级多选指标添加
- "编辑"：编辑指标权重数值，单位为%，仅支持输入数字1-100
- "删除"：鼠标移入该行显示去除按钮，点击去除该行指标
- "确定"：点击校验必填项，未填则输入框下方红色小字提示"xx不可为空！"；校验总权重是否等于100%，不等于Toast提示"指标总权重应等于100%！"；校验通过保存数据并在列表回显
- "取消"：关闭弹窗，不保存数据
- "×"：关闭弹窗，不保存数据

## 考核周期

### 需求描述
管理员可以创建考核周期，例如5月1日至5月31日为一个月度周期，还支持创建季度周期和年度周期，方便管理员进行多种方式的考核。

### 页面字段

| 字段名称 | 字段类型 | 是否必填 | 校验规则 | 说明 |
|---------|---------|---------|---------|------|
| 周期名称 | 文本 | 是 | 最大长度50字符，不允许重复 | 周期的唯一标识 |
| 周期类型 | 下拉选择 | 是 | 月度/季度/年度 | 周期的类型分类 |
| 考核开始日期 | 日期 | 是 | 必须早于考核结束日期 | 周期的起始时间 |
| 考核结束日期 | 日期 | 是 | 必须晚于考核开始日期 | 周期的结束时间 |
| 创建时间 | 日期时间 | 自动生成 | 系统自动填充 | 记录周期的创建时间 |

### 规则说明
- 有权限查看该菜单的用户可以进行考核周期维护。
- 考核计划绑定该考核周期，并且状态为"进行中"时不允许编辑。
- 有计划关联该考核周期时不允许删除。

### 交互说明

**筛选条件**：
- 创建时间：日期范围选择器

**操作栏**：
- 【新增】，点击弹出新增弹窗
- 【编辑】，点击弹出编辑弹窗，考核计划绑定该考核周期，并且状态为"进行中"时不显示该按钮
- 【删除】，点击弹出删除弹窗，如果有计划关联该考核周期，无法删除，Toast提示"该考核周期被考核计划xx关联，删除失败！"，如果被多个考核计划关联，Toast提示"该考核周期被多个考核计划关联，删除失败！"，如果未被关联，则弹窗提示

### 逻辑说明
- 【确定】，点击校验必填项，未填则输入框下方红色小字提示"xx不可为空！"；校验考核开始日期早于考核结束日期，考核结束日期晚于考核开始日期，校验不通过Toast提示；校验通过保存数据并在列表回显

## 考核计划

### 需求描述
管理员可以创建考核计划用来考核某个部门的员工，例如5月考核计划，设置考核计划的考核模板、评分人员、考核对象等，到计划开始时间后将计划推送至评分人员，由评分人员进行评分，到计划结束时间后，不允许在进行评分，在详情页可以查看详细的评分数据。

### 页面字段

| 字段名称 | 字段类型 | 是否必填 | 校验规则 | 说明 |
|---------|---------|---------|---------|------|
| 计划名称 | 文本 | 是 | 最大长度50字符，不允许重复 | 计划的唯一标识 |
| 考核周期 | 下拉选择 | 是 | 必须选择已创建的周期 | 关联的考核周期 |
| 考核模板 | 下拉选择 | 是 | 必须选择已创建的模板 | 使用的考核模板 |
| 评分人员 | 多选 | 是 | 至少选择一人 | 负责评分的人员 |
| 考核对象 | 多选 | 是 | 至少选择一人 | 被考核的人员 |
| 计划开始时间 | 日期时间 | 是 | 必须早于计划结束时间 | 计划生效的开始时间 |
| 计划结束时间 | 日期时间 | 是 | 必须晚于计划开始时间 | 计划生效的结束时间 |
| 计划状态 | 状态 | 自动计算 | 未开始/进行中/已结束 | 系统根据时间自动判断 |
| 创建时间 | 日期时间 | 自动生成 | 系统自动填充 | 记录计划的创建时间 |

### 规则说明
- 有权限查看该菜单的用户可以进行考核计划维护。
- 计划状态由系统根据当前时间自动判断：
  - 当前时间 < 计划开始时间：未开始
  - 计划开始时间 ≤ 当前时间 ≤ 计划结束时间：进行中
  - 当前时间 > 计划结束时间：已结束

### 交互说明

**筛选条件**：
- 计划名称：录入框，支持模糊查询
- 查询范围：下拉选择（全部/我创建的/我参与的）
- 计划状态：下拉选择（全部/未开始/进行中/已结束）
- 周期类型：下拉选择（全部/月度/季度/年度）

**操作栏**：
- 【新增】，点击弹出新增弹窗
- 【编辑】，点击弹出编辑弹窗，考核计划状态为未开始显示该按钮
- 【查看】，点击弹出查看弹窗，考核计划状态为"进行中/已结束"时显示该按钮
- 【删除】，点击弹出删除弹窗

### 逻辑说明
- 【确定】，点击校验必填项，未填则输入框下方红色小字提示"xx不可为空！"；校验通过保存数据并在列表回显
- 【取消/×】，关闭弹窗，不保存数据

## 考核计划结果

### 需求描述
管理员可以查看每个考核计划的考核信息和考核结果。

### 页面字段

| 字段名称 | 字段类型 | 说明 |
|---------|---------|------|
| 计划名称 | 文本 | 考核计划的名称 |
| 考核对象 | 文本 | 被考核人员姓名 |
| 绩效总分 | 数字 | 加权计算后的总成绩 |
| 各指标得分 | 表格 | 各考核指标的详细得分 |
| 评分人员 | 文本 | 负责评分的人员姓名 |
| 评分时间 | 日期时间 | 评分完成的时间 |

### 规则说明
- 有权限查看该菜单的用户可以查看考核计划的考核结果。

### 交互说明
- 点击排序按钮，进行绩效平均成绩的排序。

## 绩效评分

### 需求描述
评分人员可以在绩效评分中查看考核计划信息，对考核对象进行逐个评分，根据考核模板内的指标项进行评分，由系统加权计算绩效总分。计划完成后可以查看绩效结果。

### 页面字段

| 字段名称 | 字段类型 | 说明 |
|---------|---------|------|
| 计划名称 | 文本 | 考核计划的名称 |
| 考核周期 | 文本 | 关联的考核周期 |
| 计划状态 | 状态 | 未开始/进行中/已结束 |
| 考核对象列表 | 列表 | 待评分的被考核人员 |
| 绩效总分 | 数字 | 根据指标权重和评分实时计算 |
| 指标分类 | 文本 | 指标的所属分类 |
| 指标名称 | 文本 | 指标名称（指标权重） |
| 指标定义 | 文本 | 该指标的评分标准说明 |
| 评分 | 数字 | 当前指标的得分 |

### 规则说明
- 管理员选择的评分人员在该页面进行评分操作。
- 评分为必填项，默认为满分。
- 所有指标评分完成后才能保存。

### 交互说明

**筛选条件**：
- 查询范围：下拉选择（全部/未评/已评）

**操作栏**：
- 计划状态为"未开始"时不显示按钮
- 计划状态为"进行中"显示评分按钮
- 计划状态为"已结束"显示查看详情按钮

**绩效评分界面**：
- 左侧显示考核对象列表，未评分显示未评按钮，评分后显示已评按钮
- 右侧显示绩效总分，根据指标权重和填写的评分实时计算总分

**评分列表**：
- 指标分类、指标名称、指标定义、评分
- 指标名称显示格式：指标名称（指标权重）
- 评分默认为满分，支持手动编辑

### 交互逻辑
- 点击【保存】按钮，保存该考核对象的绩效评分成绩，如果有未填写的评分，列表中提示，Toast提示"保存失败，存在未评分项！"；保存成功后提示"保存成功！"
- 点击【返回】，返回列表页面

## 绩效结果

### 需求描述
考核对象可以在绩效结果页面查看自己在考核计划中的分数和排名。

### 页场字段

| 字段名称 | 字段类型 | 说明 |
|---------|---------|------|
| 计划名称 | 文本 | 考核计划的名称 |
| 绩效总分 | 数字 | 加权计算后的总成绩 |
| 绩效排名 | 数字 | 在所有考核对象中的排名 |
| 各指标得分 | 表格 | 各考核指标的详细得分 |
| 考核周期 | 文本 | 关联的考核周期 |

### 规则说明
- 仅考核对象可查看绩效结果菜单页面。

### 交互说明
- 绩效平均成绩支持排序。

## 总结

本PRD文档详细描述了绩效管理系统的各项功能需求，包括：
- 绩效指标管理：创建和维护考核指标
- 考核模板：组合多个指标并设置权重
- 考核周期：定义考核的时间范围
- 考核计划：组织评分人员和考核对象
- 绩效评分：评分人员进行打分
- 绩效结果：考核对象查看成绩

每个功能模块都遵循标准结构：需求描述、页面字段、规则说明、交互说明，确保开发团队能够清晰理解需求并实施。
