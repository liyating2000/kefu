## 图片列表报告格式

### 文件名
`image_list_report.json`

### JSON结构

```json
{
  "total_images": 5,
  "generated_at": "2025-03-24T06:50:00",
  "images": [
    {
      "file_name": "登录页面.png",
      "file_path": "/path/to/extracted_images/登录页面.png",
      "relative_path": "登录页面.png",
      "file_size_mb": 0.5,
      "file_extension": ".png",
      "inferred_type": "界面设计图"
    },
    {
      "file_name": "订单流程图.jpg",
      "file_path": "/path/to/extracted_images/订单流程图.jpg",
      "relative_path": "订单流程图.jpg",
      "file_size_mb": 1.2,
      "file_extension": ".jpg",
      "inferred_type": "流程图"
    }
  ]
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|-----|------|------|
| total_images | integer | 图片总数 |
| generated_at | string | 报告生成时间（ISO 8601格式） |
| images | array | 图片信息数组 |
| images[].file_name | string | 图片文件名 |
| images[].file_path | string | 图片绝对路径 |
| images[].relative_path | string | 相对于解压目录的路径 |
| images[].file_size_mb | float | 文件大小（MB） |
| images[].file_extension | string | 文件扩展名 |
| images[].inferred_type | string | 推断的图片类型 |

## 解析结果文件格式

### 文件命名规则
`{原图片名（不含扩展名）}_analysis.md`

示例：
- 原图片：`登录页面.png`
- 解析结果：`登录页面_analysis.md`

- **解析时间**: {ISO 8601时间}
- **文件大小**: {大小} MB

## PRD生成指导

### 从解析结果文件生成PRD的步骤

1. **读取汇总文件**
   - 先读取 `all_analysis_summary.md` 了解整体情况
   - 获取所有图片的概览和功能模块分类

2. **按模块组织内容**
   - 根据汇总文件中的"功能模块汇总"
   - 将相关图片的解析结果组织到同一模块

3. **读取详细解析**
   - 根据需要读取各个 `*_analysis.md` 文件
   - 提取详细的字段、交互、逻辑信息

4. **生成PRD结构**
   - 文档概述：基于汇总文件的整体描述
   - 权限说明：基于"角色权限汇总"
   - 功能模块详细说明：基于各个详细解析文件
   - 数据字段：基于"数据字段汇总"

### PRD章节与解析结果的映射

| PRD章节 | 数据来源 |
|---------|---------|
| 文档概述 | all_analysis_summary.md 的概览部分 |
| 权限说明 | 各 *_analysis.md 的"业务逻辑推断"中的权限部分 |
| 需求描述 | 各 *_analysis.md 的"解析总结" |
| 页面字段 | 各 *_analysis.md 的"数据字段和输入要求" |
| 规则说明 | 各 *_analysis.md 的"业务逻辑推断" |
| 交互说明 | 各 *_analysis.md 的"所有交互元素"和"页面跳转关系" |

## 注意事项

1. **解析完整性**: 每张图片的解析结果应尽可能详细，参考渠道维护图片的解析深度
2. **文件命名**: 严格按照命名规则，确保文件名可追溯
3. **内容一致性**: 同一模块的多张图片，术语和描述应保持一致
4. **可追溯性**: 解析结果中应保留图片路径和原始文件名，便于追溯
