## 推断维度

### 1. 功能逻辑推断

基于可见的按钮、输入框等元素，推断功能用途和操作流程。

**推断方法**：
- 按钮名称 → 推断按钮功能
- 输入框 + 标签 → 推断字段用途
- 列表 + 操作列 → 推断数据操作能力

**示例**：
```
图片解析结果：
- 按钮：「新增」
- 按钮：「编辑」
- 按钮：「删除」

推断：
该页面支持对数据的增删改操作。点击「新增」按钮打开新增表单页面，
点击「编辑」按钮打开编辑表单页面，点击「删除」按钮删除该条数据。
```

### 2. 数据流转推断

基于列表字段和操作按钮，推断数据的增删改查流程。

**推断方法**：
- 列表字段 → 推断数据结构
- 查询条件 → 推断筛选能力
- 分页组件 → 推断数据量级

**示例**：
```
图片解析结果：
- 查询条件：渠道名称、渠道id、用户体系
- 列表字段：序号、渠道名称、渠道id、用户体系、接入类型、创建时间、更新时间
- 分页：共800条记录

推断：
渠道数据支持按名称、id、用户体系进行筛选查询。数据包含渠道基本信息
和系统字段（创建时间、更新时间），总数据量约800条。
```

### 3. 权限控制推断

**重要原则**：权限控制推断需基于业务逻辑，如有疑问必须询问用户。

**推断方法**：
1. **识别敏感操作**：
   - 删除操作 → 推断是否需要删除权限
   - 配置操作 → 推断是否需要配置权限
   - 批量操作 → 推断是否需要批量操作权限
   - 权限管理功能 → 推断需要管理员权限

2. **推断权限问题**：
   - 哪些角色可以执行该操作？
   - 是否需要特殊权限控制？
   - 数据级权限还是功能级权限？

3. **有疑问时询问用户**：
   - 权限控制逻辑不明确时
   - 涉及多个角色的权限划分时
   - 数据权限边界不清晰时

**示例**：
```
图片解析结果：
- 操作按钮：编辑、配置、删除
- 功能：渠道维护

推断与提问：
基于业务逻辑，渠道维护涉及数据修改和删除，建议确认：
1. 是否所有用户都可以编辑和删除渠道？
2. 是否只有管理员可以配置渠道参数？
3. 是否需要按部门或角色限制数据访问范围？

【建议向用户提问】：
"渠道维护功能的权限控制规则是什么？是否需要区分管理员和普通用户的操作权限？"
```

**错误示例**：
```
❌ 看到账号显示ADMIN → 推断"只有管理员可访问"（未确认）
❌ 看到删除按钮 → 推断"所有人都可以删除"（可能存在风险）
✅ 看到删除按钮 → 推断"需要确认删除权限控制规则"
```

### 4. 交互流程推断

基于页面元素和跳转标注，推断用户操作流程。

**推断方法**：
- 导航结构 → 推断功能模块切换流程
- 标签页 → 推断同级功能切换
- 面包屑 → 推断页面层级关系

**示例**：
```
图片解析结果：
- 左侧导航：一级导航 + 二级导航
- 二级标签页：系统、工作组/队列、渠道维护...

推断：
用户通过一级导航选择功能大类（如网聊客服），展开二级导航选择具体功能
（如网聊维护），进入功能页面后通过二级标签页切换不同的配置模块。
```

### 5. 校验规则推断

基于字段类型和业务场景，推断常见的校验规则。**这是允许推断的内容**。

**允许推断的校验规则类型**：

1. **系统自动生成字段**
   - 创建时间、更新时间 → 推断"系统自动生成，不可修改"
   - 序号、编号 → 推断"系统自动生成，不可修改"

2. **必填字段**
   - 名称、标题类字段 → 推断"必填"
   - id、编码类字段 → 推断"必填"
   - 类型、分类类字段 → 推断"必填"

3. **长度限制**
   - 名称、标题字段 → 推断"最大长度50字符"
   - 描述、备注字段 → 推断"最大长度200字符或500字符"
   - 短文本字段 → 推断"最大长度20字符"

4. **唯一性约束**
   - id、编码字段 → 推断"系统内唯一，不允许重复"
   - 名称字段 → 推断"不允许重复"（根据业务场景）

5. **格式校验**
   - 邮箱字段 → 推断"邮箱格式校验"
   - 手机号字段 → 推断"手机号格式校验"
   - id字段 → 推断"仅支持字母、数字、下划线"

**示例**：
```
图片解析结果：
- 字段：渠道名称
- 字段：渠道id
- 字段：创建时间
- 字段：更新时间

推断：
| 字段名称 | 校验规则 |
|---------|---------|
| 渠道名称 | 必填，最大长度50字符，不允许重复 |
| 渠道id | 必填，系统内唯一标识，仅支持字母、数字、下划线 |
| 创建时间 | 系统自动生成，不可修改 |
| 更新时间 | 系统自动生成，不可修改 |
```

**推断原则**：
- ✅ 基于常见业务场景推断合理规则
- ✅ 优先推断基础校验（必填、长度、唯一性）
- ✅ 对系统字段推断自动生成规则
- ❌ 不推断过于复杂的正则规则（除非明确标注）
- ❌ 不推断数据库层面的约束（如索引、外键）

## 推断原则

### ✅ 允许的推断

1. **基于可见元素的合理推断**
   - 看到按钮 → 推断按钮功能
   - 看到输入框 → 推断输入用途
   - 看到列表 → 推断数据展示

2. **符合常见业务场景的推断**
   - 列表有查询按钮 → 推断支持筛选
   - 列表有分页 → 推断数据分页展示
   - 表单有保存按钮 → 推断数据提交保存

3. **基于常识的技术推断**
   - 表单字段 → 推断数据校验（如必填、格式）
   - 操作日志 → 推断操作记录功能
   - 缓存刷新 → 推断数据同步机制

### ❌ 禁止的推断

1. **推断图片解析结果中未提及的功能**
   - ❌ 看不到导出按钮 → 不能推断"支持导出功能"
   - ❌ 看不到批量删除 → 不能推断"支持批量删除"

2. **添加过度主观的假设**
   - ❌ 看到输入框 → 不能推断"字段有50字符长度限制"
   - ❌ 看到下拉框 → 不能推断"选项从数据库动态加载"

3. **推断未显示的技术细节**
   - ❌ 看不到接口文档 → 不能推断"接口调用方式"
   - ❌ 看不到性能指标 → 不能推断"响应时间要求"

## 正确示例

### 示例1：渠道维护页面

**图片解析结果**：
- 页面包含左侧导航栏、查询区、数据操作区、列表区、分页区
- 按钮包括：新增、编辑、配置、删除、查询、重置
- 列表字段：渠道名称、渠道id、用户体系、接入类型、创建时间、更新时间

**业务逻辑推断**：
```markdown
## 渠道维护功能需求

### 功能概述
渠道维护页面用于管理接入网聊客服的渠道信息，支持渠道的增删改查操作。

### 功能列表
1. **查询功能**：支持按渠道名称、渠道id、用户体系筛选渠道数据
2. **新增功能**：点击「新增」按钮，打开新增渠道表单页面
3. **编辑功能**：点击「编辑」按钮，打开编辑渠道表单页面，修改渠道信息
4. **配置功能**：点击「配置」按钮，进入渠道配置页面，配置渠道参数
5. **删除功能**：点击「删除」按钮，删除该渠道数据
6. **分页功能**：列表数据支持分页展示，可调整每页显示条数

### 数据字段
| 字段名称 | 字段说明 | 是否必填 | 校验规则 |
|---------|---------|---------|---------|
| 渠道名称 | 渠道的名称标识 | 是 | 最大长度50字符，不允许重复 |
| 渠道id | 渠道的唯一标识 | 是 | 系统内唯一，仅支持字母、数字、下划线，不允许重复 |
| 用户体系 | 渠道归属的用户体系 | 是 | 必须选择已存在的用户体系 |
| 接入类型 | 渠道的终端类型 | 是 | 枚举值：APP/PC/小程序/公众号 |
| 创建时间 | 渠道创建时间 | - | 系统自动生成，不可修改 |
| 更新时间 | 渠道最后更新时间 | - | 系统自动生成，不可修改 |
```

**推断说明**：
- ✅ 推断功能列表：基于可见的按钮元素
- ✅ 推断校验规则：基于字段类型推断合理规则
  - 渠道名称 → 必填、最大长度50字符、不允许重复
  - 渠道id → 必填、唯一标识、格式限制
  - 创建时间/更新时间 → 系统自动生成、不可修改
- ⚠️ 权限控制推断（需向用户提问）：
  - 新增/编辑/删除操作是否需要权限控制？
  - 配置功能是否仅限管理员？
  - 是否需要数据级权限（如部门数据隔离）？

**建议提问**：
> "渠道维护功能涉及新增、编辑、删除、配置等操作，请确认权限控制规则：
> 1. 是否所有用户都可以新增和编辑渠道？
> 2. 删除操作是否需要管理员权限？
> 3. 配置功能是否仅限特定角色？
> 4. 是否需要按部门或业务线限制数据访问范围？"

## 推断流程

在PRD生成环节，按照以下流程进行业务逻辑推断：

1. **阅读图片解析结果**：了解页面包含的所有元素和字段
2. **识别核心功能**：基于按钮和操作元素，识别主要功能点
3. **推断功能逻辑**：基于元素组合，推断功能操作流程
4. **生成需求描述**：将推断结果转化为PRD需求描述
5. **标注疑问点**：对于不确定的推断，标注为疑问并向用户提问

## 总结

业务逻辑推断是PRD生成的核心环节，但必须在合理范围内进行：

- ✅ **基础原则**：基于图片解析结果进行推断
- ✅ **合理性原则**：推断需符合常见业务场景
- ✅ **必要性原则**：推断内容需对PRD有价值
- ❌ **禁止过度推断**：不推断未提及的功能和细节
