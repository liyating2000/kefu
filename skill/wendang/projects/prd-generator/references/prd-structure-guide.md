## 概述
本文档提供产品需求文档（PRD）的标准结构和写作指南，帮助生成专业、规范的需求文档。

## 标准PRD结构

### 1. 文档概述
- **文档目的**：说明文档的作用和预期读者
- **产品背景**：产品的业务背景和市场定位
- **目标用户**：产品的目标用户群体画像

### 2. 权限说明（可选）
- **角色定义**：系统涉及的所有角色
- **权限矩阵**：各角色的功能权限范围

### 3. 功能模块详细说明
每个功能模块按以下结构描述：

#### 3.1 需求描述
- 功能的业务价值
- 用户使用场景
- 核心目标

#### 3.2 页面字段
- 字段名称
- 字段类型
- 是否必填
- 校验规则
- 默认值

#### 3.3 规则说明
- 业务规则
- 权限控制
- 数据校验规则
- 异常处理

#### 3.4 交互说明
- 用户操作流程
- 界面交互逻辑
- 按钮和操作说明（**重要**：按钮名称必须使用双引号包裹，格式为：`- "按钮名"：描述`，例如：`- "编辑"：点击弹出编辑产品弹窗`）
- 反馈提示

#### 3.5 逻辑说明（可选）
- 复杂业务逻辑
- 算法规则
- 计算公式

### 4. 非功能需求（可选）
- 性能要求
- 安全要求
- 兼容性要求
- 可用性要求

### 5. 附录（可选）
- 术语表
- 参考资料
- 版本历史

## 各章节写作要点

### 需求描述写作要点
- **明确性**：使用简洁明了的语言，避免模糊表述
- **场景化**：从用户角度描述使用场景，而非技术实现
- **价值导向**：说明功能如何解决用户问题或带来业务价值

**示例**：
```
管理员想要对部门成员绩效考核时，需要先创建考核的指标。点击"指标管理"菜单，
进入绩效指标维护界面。界面可实现对绩效指标信息的增、删、改、查功能。
```

### 页面字段写作要点
使用表格形式呈现字段信息，清晰展示字段属性：

| 字段名称 | 字段类型 | 是否必填 | 校验规则 | 说明 |
|---------|---------|---------|---------|------|
| 指标名称 | 文本 | 是 | 最大长度50字符，不允许重复 | 考核指标的唯一标识 |
| 指标权重 | 数字 | 是 | 1-100之间的整数 | 该指标在考核中的权重占比 |
| 指标定义 | 文本 | 是 | 最大长度500字符 | 详细说明该指标的评分标准 |

### 规则说明写作要点
- **权限规则**：明确哪些角色可以执行哪些操作
- **数据规则**：数据的创建、更新、删除条件
- **校验规则**：输入数据的格式和范围限制
- **异常处理**：异常情况的处理方式

**示例**：
```
- 有权限查看该菜单的用户可以进行指标维护
- 指标名称在系统中必须唯一，重复时提示"指标名称已存在"
- 指标被考核模板引用后，不允许删除
```

### 交互说明写作要点
按用户操作流程描述，使用步骤化表述。

**重要格式规范**：按钮和操作项必须使用双引号包裹，格式为：`- "按钮名"：描述`

**示例**：
```
**操作按钮**：
- "新增"：点击弹出新增弹窗
- "编辑"：点击弹出编辑弹窗，修改后保存
- "删除"：点击弹出删除确认弹窗，确认后删除

**弹窗操作**：
- "确定"：点击校验必填项，未填则输入框下方红色小字提示"xx不可为空！"
- "取消"：关闭弹窗，不保存数据
- "×"：关闭弹窗，不保存数据
```

**错误示例（禁止使用）**：
```
- **新增**：点击弹出新增弹窗  ❌
- **编辑**：点击弹出编辑弹窗  ❌
```

### 逻辑说明写作要点
针对复杂业务逻辑，使用流程化描述：

**示例**：
```
【指标权重】，默认无数据需要手动填写，所有指标的总权重为100%
- 校验总权重是否等于100%，不等于Toast提示"指标总权重应等于100%！"
- 校验通过后保存数据并在列表回显
```

## 写作规范

### 语言风格
- 使用祈使句和陈述句，避免疑问句
- 使用第三人称，避免第一人称
- 术语统一，避免同义词混用

### 格式规范
- 标题层级清晰，不超过3级
- 列表项使用统一的符号（- 或 1. 2. 3.）
- 表格用于结构化数据展示
- 重要信息使用**加粗**强调

### 描述规范
- 操作描述：使用"点击"、"选择"、"输入"等明确动词
- 状态描述：使用"显示"、"隐藏"、"启用"、"禁用"等明确状态
- 提示信息：完整写出提示内容，用引号或代码格式标识

### 避免的问题
- 避免技术实现细节（如数据库设计、接口定义）
- 避免模糊表述（如"合理的"、"适当的"、"优化的"）
- 避免冗余描述（如重复说明相同规则）
- 避免遗漏关键信息（如错误提示、边界条件）
