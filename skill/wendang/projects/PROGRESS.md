# PRD Generator Skill - 开发进度管理

## 版本历史

### v2.2.0 (2025-03-24)
**状态**: 已完成

**核心原则**:
- 新增"主动提问原则"：整个PRD生成过程中遇到任何疑问必须向用户提问
- 将"主动提问原则"置于注意事项首位，强调重要性

**提问环节新增位置**:

1. **步骤1.4 图片解析确认环节**：
   - 图片内容不清晰或模糊
   - 多张图片之间存在矛盾
   - 无法确定优先级或重要性
   - 不清楚专业术语或业务场景
   - 无法推断交互逻辑或业务规则

2. **步骤2 需求分析环节**（最重要）：
   - 业务场景不清晰（使用场景、目标用户）
   - 功能优先级不确定（核心功能vs辅助功能）
   - 业务规则不明确（校验规则、异常处理、权限控制）
   - 数据流转不清晰（数据来源、去向、生命周期）
   - 技术实现疑问（系统对接、性能要求、安全要求）
   - 内容遗漏或矛盾（未标注区域、描述不一致）
   - **提问方式**：一次提出所有疑问，提供选项，说明影响

3. **步骤3 PRD生成前**：
   - 文档目标读者
   - 详细程度要求
   - 是否包含非功能需求
   - 特定格式要求或模板
   - 技术实现建议

4. **步骤3 PRD生成后**：
   - 询问用户是否符合需求
   - 是否需要调整格式、补充内容或修改章节

5. **步骤4 质量检查后**：
   - 缺少关键信息
   - 业务逻辑存在潜在矛盾
   - 权限控制不够清晰
   - 异常处理机制不完整

**关键改进**:
- 明确"不要猜测或假设"的原则
- 提供提问方式指导：一次提出、提供选项、说明影响
- 在每个关键步骤都设置疑问澄清环节
- 确保PRD的准确性和完整性

### v2.0.0 (2025-03-24)
**状态**: 已完成

**核心功能**:
- 支持批量图片处理：ZIP包解压、逐张解析、独立文件保存
- 每张图片生成独立的解析结果文件（`*_analysis.md`）
- 生成解析结果汇总文件（`all_analysis_summary.md`）
- 按功能模块组织多张图片的解析结果

**新增资源**:
1. **脚本工具**：
   - `scripts/batch_image_processor.py`：批量图片处理器
   - 功能：解压ZIP、扫描图片、生成图片列表报告

2. **参考文档**：
   - `references/batch-image-format-guide.md`：批量图片解析格式规范
   - 定义了图片列表报告、解析结果文件、汇总文件的标准格式

**流程优化**:
- 步骤1重新设计为"批量图片处理与解析"：
  - 子步骤1.1：调用脚本解压ZIP并获取图片列表
  - 子步骤1.2：逐张调用read_image解析图片
  - 子步骤1.3：将解析结果保存到独立文件
  - 子步骤1.4：生成解析结果汇总文件
  - 子步骤1.5：输出汇总结果供用户确认
- 步骤2：读取解析结果文件进行需求分析
- 步骤3：整合所有解析结果生成PRD

**解决痛点**:
- 解决ZIP包图片过多导致解析不准确的问题
- 每张图片单独解析，确保解析深度和准确性
- 参考渠道维护图片的解析效果，提供详细的解析结果

**文件结构**:
```
./
├── extracted_images/          # 解压的图片
│   ├── 渠道维护.png
│   ├── 员工管理.png
│   └── ...
├── image_analysis_results/    # 解析结果
│   ├── 渠道维护_analysis.md
│   ├── 员工管理_analysis.md
│   └── all_analysis_summary.md
├── image_list_report.json     # 图片列表报告
└── prd_document.md           # 最终PRD
```
